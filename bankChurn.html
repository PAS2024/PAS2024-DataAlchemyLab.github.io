Bank Churn dataset

Problème

Le jeu de données met en évidence le défi de la rétention client et la gestion du churn (désabonnement). Des facteurs tels que le score de crédit, le solde bancaire, l’ancienneté ou l’engagement du client peuvent influencer la décision de quitter la banque. Ne pas anticiper ces comportements peut entraîner une érosion des revenus et de la base client.

Solution

Ce modèle analyserait des variables clés (comme le solde, le nombre de produits utilisés et l’engagement) pour anticiper les départs. L’objectif est de mettre en place des actions ciblées comme des offres personnalisées, augmentant ainsi la fidélité. L’utilisation de Power BI permet de visualiser ces tendances, facilitant la prise de décision éclairée pour améliorer la rétention client.

Dataset information

Ce jeu de données contient des informations démographiques et bancaires sur les clients, incluant :

  • CustomerID : Identifiant unique pour chaque client.
  • Surname : Nom de famille du client.
  • CreditScore : Score de crédit numérique du client.
  • Geography : Pays de résidence (France, Espagne ou Allemagne).
  • Gender : Sexe du client (Homme ou Femme).
  • Tenure : Nombre d'années de fidélité du client à la banque.
  • NumOfProducts : Nombre de produits bancaires utilisés par le client (par exemple, compte épargne, carte de crédit).
  • HasCrCard : Indique si le client possède une carte de crédit (1 = oui, 0 = non).
  • IsActiveMember : Indique si le client est un membre actif (1 = oui, 0 = non).
  • EstimatedSalary : Salaire estimé du client.
  • Exited : Indique si le client a quitté la banque (1 = oui, 0 = non).

Outils

While our project had a rather audacious scope at the beginning, the scope has been reduced as we have developed the system as well as learned more about the domain. Specifically, the project focused on supporting situations where accurate plant data collection and environmental maintenance is critical, such as research, commercial or experimentation applications. This is not to say that these are the only situations under which the system could operate, but through our user research we found these to be the most applicable use cases for such a system.
